技术总览Technical Overview

自回归（AR）语言模型将文本生成建模为链式条件概率的逐步采样；扩散语言模型（dLM）则将其建模为从全掩码序列到目标序列的迭代去噪过程。两者的根本区别在于：前者是严格的顺序生成，后者允许全局建模与并行揭示。 Autoregressive LMs decompose text generation as a chain of conditional probabilities; diffusion LMs model it as iterative denoising from a fully masked sequence. The former emphasizes sequential generation, the latter global modeling and parallel reveal.

问题定义Problem

语言生成能否摆脱固定的从左到右的顺序约束，转而优先确定高置信度位置？Can language generation move beyond fixed left-to-right order and prioritize high-confidence positions?

核心答案Core Answer

dLM 以随机掩码、双向预测与多步揭示，替代 AR 的单向 next-token prediction。DLLM replaces AR's unidirectional next-token prediction with random masking, bidirectional prediction, and multi-step reveal.

工程意义Engineering Impact

其意义不在于替代 AR，而在于扩展生成范式：天然支持 infilling、全局重评估与可控的揭示顺序。Rather than simply replacing AR, it extends the generation paradigm: naturally supporting infilling, global re-evaluation, and controllable reveal order.

AR 的本质：一条链AR Essentials: A Chain

p(x) = ∏i=1N p(xi | x<i)

文本被分解为一条严格有序的决策链：第 i 个词元仅依赖其左侧上下文，信息流严格单向。 Text is decomposed into a strictly ordered decision chain: token i depends only on left context; information flows unidirectionally.

  • 训练与推理的目标完全一致：均为 next-token prediction。Training and inference share the same objective: next-token prediction.
  • 每一步仅针对当前位置，缺乏全局重评估机制。Each step targets only the current position with no global re-evaluation.
  • 生成顺序固定为从左到右，逐词元展开。Generation order is fixed: left-to-right, token by token.

为什么需要 dLMWhy DLLM

串行生成 O(N)Sequential O(N)

生成 N 个词元需要 N 次前向传播。KV Cache 仅能降低单步开销，无法改变串行的本质。Generating 1000 tokens requires ~1000 forward passes. KV cache reduces cost but not the sequential nature.

固定顺序的局限性Fixed-Order Limitation

许多任务天然适合先确定全局结构、再填充局部细节的生成方式。Many tasks benefit from determining global structure first, then filling in details.

Exposure Bias

训练依赖 ground truth 而推理依赖模型自身输出，二者的分布偏差随步数累积。Training relies on ground truth while inference relies on model output; errors accumulate.

顺序生成范式（AR）Sequential Paradigm (AR)

严格从左到右逐词元生成；一旦某位置出错，后续所有条件分布均受污染。Strictly left-to-right; once an error occurs, all subsequent conditional distributions are affected.

全局揭示范式（dLM）Global Reveal Paradigm (DLLM)

基于全局上下文同时评估所有位置的置信度，优先填充最确定的位置；揭示顺序由模型自行决定。Evaluates confidence across all positions globally; fills highest-confidence positions first; order is dynamically determined.

从连续扩散到离散扩散From Continuous to Discrete Diffusion

图像扩散使用连续高斯噪声；文本不存在自然的连续噪声空间，因此 dLM 以 MASK token 作为离散噪声的替代。 Image diffusion uses continuous Gaussian noise; text has no natural continuous noise space, so DLLM uses MASK as a discrete noise proxy.

和 BERT / AR 的关键区别Key Differences from BERT / AR

相较于 BERTvs. BERT

BERT 使用固定的 mask 比例（如 15%）；dLM 通过连续参数 t 覆盖从低到高的全部损坏强度。BERT uses a fixed mask ratio; DLLM uses continuous t covering all corruption intensities.

相较于 ARvs. AR

AR 仅学习预测下一个词元；dLM 学习同时恢复多个被掩码位置。AR learns only the next token; DLLM learns to recover multiple positions simultaneously.

采样与控制：Gumbel-Max、Top-k、CFGSampling & Control: Gumbel-Max, Top-k, CFG

Gumbel-Max 采样Gumbel-Max Sampling

相较 softmax + multinomial，Gumbel-Max 在低精度数值条件下更稳定，适用于推理阶段的离散采样。More stable than softmax + multinomial under low precision; ideal for discrete sampling during inference.

temperature

控制采样随机性：0 趋近贪心解码，1 为标准采样。过高则输出散乱，过低则重复。Controls sampling randomness: 0 → greedy, 1 → standard; too high → incoherent, too low → repetitive.

Top-k 与 CFGTop-k & CFG

Top-k 截断长尾噪声；CFG 通过有条件与无条件两次前向传播的差值来增强 prompt 引导。Top-k truncates long-tail noise; CFG amplifies prompt constraint via conditional/unconditional forward difference.

结论并非“dLM 已取代 AR”，而是：dLM 将语言生成从严格的顺序链式过程，扩展为可迭代、可全局重评估的推理过程。 The conclusion is not “DLLM replaces AR”, but: DLLM extends language generation from a strict sequential chain into an iterative, globally re-evaluable inference process.

调参建议Tuning Guidelines

block_size
8–32

值越小越稳定但越慢，越大越快但对步数与采样策略更敏感。Smaller = slower but stabler; larger = faster but depends more on steps & sampling.

steps
≥ block_size

步数过少会导致每步揭示过多，补全质量下降。Fewer tokens revealed per step; too few steps leads to coarse completion.

temperature
0.2–0.8

过高则输出散乱，过低则重复；通常需与 top_k 联合调节。Too high = chaotic; too low = repetitive; usually tuned with top_k.

top_k
20–100

截断候选词元空间，平衡多样性与生成稳定性。Controls candidate token space; affects diversity and stability.

cfg_scale
0.5–2.0

增强对 prompt 的忠实度，但过高可能引发模式崩塌。Improves prompt fidelity; too high may cause mode collapse.

总结Summary

AR 基于链式分解逐词元生成；dLM 基于全局建模迭代揭示。AR generates token by token via chain decomposition; DLLM reveals iteratively via global modeling.

二者并非简单的替代关系：AR 强调顺序一致性，dLM 强调全局建模与迭代修正。 They are not simple substitutes. AR emphasizes sequential consistency; DLLM emphasizes global modeling and iterative refinement.
